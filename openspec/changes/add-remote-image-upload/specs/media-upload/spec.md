## 新增需求
### 需求：远程图片选择入口
系统必须在图片上传组件中提供“远程选择”入口，允许用户从历史图片中选择。

#### 场景：打开远程选择弹框
- **当** 用户选择远程图片上传入口
- **那么** 系统弹出图片选择弹框
- **并且** 弹框请求 `GET /api/media/feed?mediaTypes=IMAGE&limit=6`

#### 场景：展示图片列表
- **当** `/api/media/feed` 返回图片数据
- **那么** 弹框每行展示 3 张图片
- **并且** 仅展示最近 6 条记录

#### 场景：选择图片写入
- **当** 用户在弹框中选择某张图片
- **那么** 系统必须记录该图片的 `fileUuid`
- **并且** 与本地上传生成的输入结构保持一致

### 需求：生成请求携带远程图片
系统必须在生成请求中携带远程选择的 `fileUuid`，与本地上传逻辑一致。

#### 场景：提交生成
- **当** 用户已选择远程图片并点击生成
- **那么** 请求参数中包含对应的 `fileUuid`
- **并且** 后端处理路径与本地上传保持一致
