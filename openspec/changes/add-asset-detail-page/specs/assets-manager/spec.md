## 新增需求
### 需求：资产详情页面入口
系统必须提供资产详情页面，并允许从资产列表进入详情页面。

#### 场景：从资产列表进入详情页
- **当** 用户在资产管理列表中点击某个资产卡片的详情入口
- **那么** 系统跳转到 `/{locale}/assets/{taskId}` 详情页面

### 需求：资产详情页面 SSR 展示
系统必须以 SSR 方式渲染资产详情页面并保持 SEO 友好。

#### 场景：登录用户访问详情页
- **当** 登录用户访问 `/{locale}/assets/{taskId}`
- **那么** 系统调用 `GET /api/media/result/{taskId}` 获取详情数据
- **并且** 以服务端渲染输出页面内容

### 需求：全媒体类型详情预览
系统必须支持图片、视频、音频三种媒体类型的详情预览。

#### 场景：展示图片资产
- **当** 详情数据为图片类型
- **那么** 系统展示图片预览

#### 场景：展示视频资产
- **当** 详情数据为视频类型
- **那么** 系统展示视频预览

#### 场景：展示音频资产
- **当** 详情数据为音频类型
- **那么** 系统展示音频播放器与封面（若存在）

### 需求：详情信息 Tab 分组
系统必须以 Tab 形式展示详情信息，并仅展示存在的数据字段。

#### 场景：展示可用字段
- **当** 详情数据包含字段（如 prompt、模型、分辨率、时长、状态、生成时间、参数等）
- **那么** 系统在对应 Tab 中展示该字段
- **并且** 对于缺失字段不显示占位内容

### 需求：未登录访问提示
系统必须在用户未登录时提示登录，不请求详情数据。

#### 场景：未登录访问详情页
- **当** 未登录用户访问 `/{locale}/assets/{taskId}`
- **那么** 页面显示登录提示
- **并且** 不调用详情接口

### 需求：资产不存在提示
系统必须在详情数据为空或不存在时给出明确提示。

#### 场景：资产不存在
- **当** `GET /api/media/result/{taskId}` 返回空数据或 404
- **那么** 页面显示 "Asset not found"
